Binary Search 二分查找

  • **Java**基本实现

  • 时间复杂度

    • 按时间复杂度从低到高
      • $O(1)$ 常量时间 意味着算法时间并不随数据规模而变化
      • $O(log(n))$ 对数时间
      • $O(n)$ 线性时间 算法时间与数据规模成正比
      • $O(n*log(n))$ 拟线性时间
      • $O(n^2)$ 平方时间
      • O$(2^n)$ 指数时间
      • $O(n!)$ 阶乘时间
    • 渐进上界asymptotic upper bound：从某个常数$n_0$开始， $c*g(n)$总是位于$f(n$)上方，那么记作$O(g(n))$,代表算法执行的最差情况
    • 渐进下界asymptotic lower bound：从某个常数$n_0$开始， $c*g(n)$总是位于$f(n$)下方，那么记作$\Omega(g(n))$,代表算法执行的最好情况
    • 渐进紧界asymptotic tight bound：从某个常数$n_0$开始，$f(n)$总是位于$c_1g(n)$和$c_2g(n)$之间，那么记作$\Theta(g(n))$
    • 二分查找
      • 时间复杂度：最坏：$O(log(n))$、最好$O(1)$
      • 空间复杂度：常数个指针$i,j,m$,额外占用空间是$O(1)$

  • **LeftRightmost**

    • 应用

    • code

      • 求排名rank：leftmost(n)+1
      • 求前任predecessor：leftmost(n)-1
      • 求后任successor：rightmost(n)+1
      • 最近邻居

    • 范围查询

      • $x \lt 4$ —> $0 .. leftmost(4) - 1$
      • $x \leq 4$ —> $0 .. rightmost(4)$
      • $4 \lt x$ —> $rightmost(4) + 1 .. \\infty$
      • $4 \leq x$ —> $leftmost(4) .. \\infty$
      • $4 \leq x \leq 7$ —> $leftmost(4) .. rightmost(7)$
      • $4 \lt x \lt 7$ —> $rightmost(4)+1 .. leftmost(7)-1$
